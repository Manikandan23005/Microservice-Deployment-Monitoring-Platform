# AI Incident Analysis

## Overview
This document describes the design of the pluggable AI Incident Analysis engine in the DevOps Nexus platform.

## Goals
- Support pluggable AI backends (local Ollama containers, OpenAI APIs, Groq endpoints, LM Studio targets).
- Automate cluster fault diagnostic generation without locking down vendor dependencies.

## Implementation Plan
1. **Model Selection Configurations:**
   - Define config environment variables (`AI_PROVIDER`, `OLLAMA_HOST`, `OPENAI_API_KEY`, etc.) inside `.env.example`.
2. **Abstract AI Service Interface:**
   - Implemented in `app/services/ai_engine.py` using abstract interfaces.
   - Dynamically initializes the client wrapper (e.g. OpenAI SDK, Ollama API post payload calls).
3. **Retrieval Augmented Generation (RAG):**
   - Correlates alert signatures, Loki log outputs, and deployment git commit metadata to draft diagnosis suggestions.

## Future Work
* **Local Ollama Model Installer:** Automatically fetch required models (such as `codellama` or `mistral`) during setup.
* **Vector Store Integrations:** Index local system runbooks in pgvector to provide contextual resolution recipes.
