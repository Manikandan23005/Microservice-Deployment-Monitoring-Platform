# AI Incident Assistant

This document details the pluggable AI completion engine integrations.

## Supported Providers
The AI engine client wrapper supports four pluggable backends:
1. **Ollama:** Configured via `OLLAMA_HOST` targeting local model completions (default model `llama3`).
2. **OpenAI:** Remote completions using `OPENAI_API_KEY` (model `gpt-4-turbo`).
3. **Groq:** High-speed cloud model completions using `GROQ_API_KEY` (model `llama3-8b-8192`).
4. **LM Studio:** OpenAI-compatible local endpoints using `LMSTUDIO_HOST`.

## Triage Prompts
- **Chat completions:** General conversation queries for cluster troubleshooting.
- **Incident analysis:** Combines active pod events details and logs to build structured Markdown reports detailing the Incident Summary, Root Cause Analysis, and Remediation Commands.
